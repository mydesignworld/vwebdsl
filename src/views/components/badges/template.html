Standard Badges are primarily used to both seperate and highlight more discrete pieces of information, so that they are both brought to the users attention but not in a manner that will add to or overload cognition. Statuses Badges highlight content that where a user needs a way to clearly understand progres. They should be clearly differentiated by colour and icon treatments.

A badge should be compact and represent discrete information only.
Statuses should be easy to find among other components

A badge should help highlight information and be clear and easy to scan. Different styles inform users of different status levels

Should have a clearly understood relationship to the content it supports
Name Type Description Accepted
primary Boolean Will set badge primary L&F state false, true
secondary Boolean Will set badge secondary L&F state false, true
dark Boolean Will set badge dark background L&F state false, true
light Boolean Will set badge light (no) background L&F state false, true
state String Different states the badge can be in. It will change the badge's color default, caution, success, warning
size String Will determine the badge size normal, small, xsmall
icon-left String Will add any of our predefined icons to the left side of the badge. E.g. icon-left="bi-dash-circle" icon-left=""
icon-right String Will add any of our predefined icons to the right side of the badge. E.g. icon="bi-dash-circle" icon-right=""
