A drop down displays a list of choices on a temporary surface. They appear when users interact with a button, action, or other control.

Drop downs should be clearly visible, easy to open, close, and interact with.

The content of a drop down should be suited to user needs.

Items within a drop down should be understandable and easy to scan.

Each option for Drop Downs must follow the following structure:

Property Type Description
id String Option unique identifier
value String Option value
label String Option text to display
disable Boolean or Function Whether this option is disabled or not.
In case of being a Function, its interface is function({Object model}) {}
Name Type Description Accepted
:options Array Will populate the options within the Drop Down/Select.
This is required.		 See option's model description above.
size String Will determine the drop-down size normal, small
defaultValue String Will determine the default value of the selected option as well as the selected value. E.g. defaultValue="Select Account" Select Option, "Any String"
disabled Boolean Will set drop-down to disabled state with no functionality false, true
model Object Associated reference to the drop-down selector
id String Drop Down unique DOM identifier
name String Drop Down unique form identifier
