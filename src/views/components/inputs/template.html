Inputs fields enable the user to interact with and input content and data, usually to enter and edit text. They use a container or area to provide a clear affordance for interaction, making these fields easily discoverable in any given layout. They should be:

Input fields should create contrast between themselves and surrounding content

Input field states should be clearly differentiated from one another

Input fields should make it easy for users to understand the information required to populate
Name Type Description Accepted
labels Boolean If neither labels, lines or titles are specified lables will be the default. Will set input to label L&F state false, true
lines Boolean Will set input to line L&F state false, true
type String Different types that are allowed for input fields text, password, email, number
label String Will add a label for the input. E.g. label="username" icon=""
id String Will add an identifier for the input, and bind it to the label. If no id is evident the lable will be used as an id. E.g. id="username" icon=""
errorMessage String Will add message that should display as error on validation fail. E.g. errorMessage="Not a valid username" icon=""
placeholder String To add a placeholder description to the input, but also to assit with animating labels. E.g. placeholder="Please enter username" icon=""
value String The value of the input to submit. can either be String or Number. icon=""
disabled Boolean Will set input to disabled state with no funtionality false, true
required Boolean Will make input mandatory before submit can happen false, true
